Download by Platforms & Versions

Earlier versions of E have been tested and run on MSWindows (95, 98 FE, 98 SE, NT, 2K), Linux, Solaris, FreeBSD, and now Mac OS X (which is really a FreeBSD platform with its own GUI toolkit and JDK). It should run on other UNIX platforms as well, given an adequate version of Java and bash (see below). It should also run without problems on MSWindows ME, but as far as we are aware, no one has tried this. If you experience any problems, or have any other informative experiences, please report a bug or post to the e-lang list.

The Installing links below describe how to install, and run various forms of the binary distribution. The Building links describe how to build E from the source release. The Download links will download each corresponding form of the release to your machine. (If a binary distribution isn't available for your platform, you can use the "purej" variant, which does not include SWT, or download and build the source distribution. If you have any problems building from the source distribution, please let us know.)

Dependencies

Versions & Types of Java

E requires a JDK compatible with the Sun JDK >= 1.4. Because E's CapTP implementation currently uses Java serialization, due to the calendar security bug, we strongly discourage use of a JDK <= 1.4.2_18, <= 5u16, or <= 6u10. For all platforms other than the Mac, adequate JDKs are available from java.sun.com. As of this writing, no secure Java is available for the Mac. Note that this problem does not affect Joe-E or Waterken.
