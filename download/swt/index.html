Adding SWT

SWT is the Standard Widget Toolkit from IBM's Eclipse project, and, starting with E 0.8.20, the new UI toolkit for E. E will continue to support Swing, but we expect to increasingly support and emphasize SWT as we increasingly deemphasize Swing. In particular, due to its size, we don't expect to ever properly tame Swing, and so locally untrusted code (including all caplets) can never be given access to the Swing API.

Unfortunately, there is an SWT per-platform, and some of these have their own installation issues. This prevents us from bundling SWT in with all E distributions, as we could not test installation on platforms we have no access to. Rather, we bundle it in with a few selected distributions, and make an SWT-less distribution so you can try combining E with SWT on your platform yourself. Please let us know of the results.

The official distribution site for SWT is http://www.eclipse.org, so you should go there to obtain the official and current versions of the SWT files we make available here. Go to downloads, pick a download server, pick a recent good release, go to the bottom of the page for SWT, and pick an SWT for your platform, such as MSWindows.
